# Reportage VSCode Extension

[![License](https://img.shields.io/badge/License-Apache_2.0-blue.svg)](https://opensource.org/licenses/Apache-2.0)

Syntax highlighting and code snippets for [reportage](https://github.com/tooppoo/reportage) test files.

## What is Reportage?

Reportage is a declarative test framework for command-line tools and shell scripts.
It lets you write test cases that specify commands to run, files to create, and assertions to verify behavior.

## Features

**Syntax Highlighting**
Full syntax highlighting for `.repor` files with support for all reportage constructs: cases, actions, assertions, and operators.

**Code Snippets**
30+ snippets to accelerate writing test cases, including templates for common patterns like exit codes, file assertions, and output validation.

**Language Support**
Embedded shell syntax highlighting for action steps and automatic bracket matching and auto-closing pairs.

## Quick Start

1. Install the extension from the VSCode marketplace
2. Open or create a `.repor` file
3. Start typing and use snippets to scaffold test cases

### Operators

- `not` — Negate the group result of the child expectations as a whole

  ````
  not {
    exit 0
  }
  ````

- `all` — Pass only when every child expectation passes

  ````
  all {
    exit 0
  }
  ````

- `any` — Pass when at least one child expectation passes

  ````
  any {
    exit 0
  }
  ````

The examples below are taken from [examples/full-syntax.repor](examples/full-syntax.repor), which is run by reportage's own e2e suite, so they always reflect working syntax.

````reportage
case "actions and write steps" {
  $ printf 'one\ntwo\nthree\n' \
    | grep --color=never 'two' \
    | tr 'a-z' 'A-Z'

  write <"greeting.txt"> "hello\n"

  write <"quoted.txt"> "she said \"hi\" and used a \\ backslash\n"

  write <"report.md"> ```
    # Report
    Indentation here is dedented relative to the closing fence,
    so the body can be indented without affecting its content.
    ```

  assert {
    exit 0 # trailing comment example
    stdout contains "TWO"
    file <"quoted.txt"> text_equals "she said \"hi\" and used a \\ backslash\n"
  }
}
````

````reportage
case "expectations across all kinds and literal forms" {
  $ printf 'hello\n' > actual.txt
  $ printf 'hello\n' > expected.txt
  $ printf 'hello\n'; printf 'hello\n' 1>&2

  assert {
    exit 0
    stdout contains "hello"
    stdout contents_equals <"expected.txt">
    stdout text_equals "hello\n"
    stderr contains "hello"
    stderr text_equals ```
    hello
    ```
    file <"actual.txt"> exists
    file <"actual.txt"> contains "hello"
    file <"actual.txt"> contents_equals @"fixtures/expected.txt"
    file <"actual.txt"> text_equals ```
    hello
    ```
  }
}
````
